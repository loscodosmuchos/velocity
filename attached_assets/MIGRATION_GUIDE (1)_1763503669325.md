## âš ï¸ Special Notes and Caveats

### Migration Pitfalls
1. **Database Connection Strings**: Different providers use different formats
   - PostgreSQL: `postgresql://user:pass@host:port/db`
   - Neon: `postgresql://user:pass@host/db?sslmode=require`
   - Supabase: `postgresql://postgres:pass@host:port/postgres`

2. **File Path Handling**: 
   - Replit uses absolute paths starting from `/`
   - Other platforms may require relative paths
   - Update asset imports in `vite.config.ts` alias section

3. **Environment Variables**:
   - Replit auto-provides some variables
   - Other platforms require manual configuration
   - Always use `.env.example` for reference

### Security Considerations
```bash
# Essential security measures
1. Never commit API keys to version control
2. Use environment variables for all secrets
3. Enable CORS properly for production domains
4. Implement rate limiting for API endpoints
5. Validate all user inputs with Zod schemas
6. Use HTTPS in production
7. Implement proper session management
```

### Known Issues
- **Port conflicts**: Ensure ports 5000 and 5173 are available
- **Database connections**: Check PostgreSQL is running and accessible
- **File permissions**: Ensure the app can write to temp directories
- **Node.js versions**: Use Node.js 18+ for optimal compatibility

